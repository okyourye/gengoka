# 言語化トレーニングアプリ 検証手順書

## 🚀 アプリの起動方法

ターミナルで以下のコマンドを実行し、開発サーバーを起動してください。

```bash
cd gengoka
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) にアクセスします。
スマートフォンでの確認には、Chromeのデベロッパーツール（Cmd+Opt+I -> モバイルアイコン）をご利用ください。

## 🧪 確認シナリオ

### 1. トップページとアニメーション
- [ ] ページロード時にタイトルやボタンがスムーズにフェードインすることを確認。
- [ ] **レイアウト確認**: ヘッダーがなく、上部が詰まって表示されていること。
- [ ] 「トレーニングを始める」ボタンをクリックして、トレーニング画面へ遷移すること。

### 2. Step 1: 思考（どう思う？）
- [ ] **表示確認**:
    - テーマ「理想の上司に必要なことは何か？」が表示されているか。
    - 「どう思う？どう感じる？」という質問が表示されているか。
    - タイマーが2分からカウントダウンしているか（※ ステップ2と合計で2分）。
    - 右側の「ヒント & 実例」セクションが表示されているか（**モバイル時は下部**）。
- [ ] **動作確認**:
    - 入力欄にテキストを入力し、Enterキーを押すと新しい行が追加されること。
    - 行と行の間に「それってどういうこと？」というガイドが表示されること。
    - 「ヒント & 実例」をクリックしてアコーディオンが開閉すること。
    - 「次へ進む」ボタンで次のステップへ遷移すること（タイマーはリセットされない）。

### 3. Step 2: 理由（なぜそう思う？）
- [ ] **表示確認**:
    - 「なぜそう思う？」という質問に変わっているか。
    - 直前のステップの最後の入力（最も解像度が高い部分）が「あなたの考え」として引用表示されているか。
- [ ] **動作確認**:
    - 理由を入力できること。同様にEnterキーで深掘りできること。
    - 「完了する」ボタンでレビュー画面へ遷移すること。

### 4. 結果・レビュー
- [ ] **表示確認**:
    - 「トレーニング完了」画面が表示されること。
    - ステップ1（思考）とステップ2（理由）の入力内容が並んで表示され、振り返りができること。
- [ ] **動作確認**:
    - 「トップへ戻る」でトップページに戻れること。
    - 「もう一度行う」で最初からやり直せること。

## 📱 レスポンシブ確認 (推奨: iPhone SE/12/14/Pixel等)
- [ ] **モバイル表示**:
    - メインコンテンツが幅375px程度でも崩れずに表示されること。
    - 「ヒント機能」がメインコンテンツの下に配置され、アコーディオンとして機能すること。
    - タイマーとフェーズ表示が適切に画面内に収まっていること。
- [ ] **インタラクション**:
    - モバイルでも「深掘り入力（Enterキー）」や「次へ」ボタンが押しやすいこと。
